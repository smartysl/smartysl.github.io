通过观察发现，在最小化的浏览器显示下，所有元素都按照从上到下的顺序依次排列。浏览器放大后夹心饼的图片浮动到了右侧，ingredients所在的标签也浮动到了methon标签所在位置的右侧，这时就要使用@media语法(媒体查询)来实现这个功能

媒体查询结构(例)：


@media only screen and (min-width:400px) {
         body { background-color:blue;}
}

以@media开头来表示这是一条媒体查询语句。紧跟在@media后面的是一个或者多个表达式，可以判别为真或假。在创建媒体查询时，必须以媒体类型（此处为screen）作为第一个表达式：@media only screen。与媒体类型相对应的有：print（打印机）、braille（盲文）或者all（全部）等其他类型。and是一个关键字，表示有多个表达式，screen是第一个表达式，and后为另一个表达式。需前后表达式都为真时，整条查询结果才为真（也就是“且”的意思）。与and相对应的关键字有：or、not等。and之后的(min-width:400px)为第二个表达式（表达式在小括号内），所有的表达式都应包含在括号内，除非它是只有一个单词的媒体类型表达式（如：screen、print等）。前缀min-表示“至少”，即“大于等于”的意思。前缀max-表示“至多”，即“小于等于”的意思。{}中的内容就是按照我们需要实现的功能设置不同标签的行为了(如body的背景色改为蓝色)

响应式设计的思路应该为：从最小界面开始设计

代码设计的整体思路就是将需要响应的元素和相关元素使用div标签进行包裹，以便对元素进行相关的位置操作。这个网页设计还是比较简单的(代码随手写的没有优化)，唯一新学的知识是display属性：


p.inline
  {
  display:inline;
  }

这种写法可以去掉p标签后的换行符，实现p标签与img标签同行排列
优化的方法主要是使用流式布局(将网页从固定布局修改为百分比布局)
